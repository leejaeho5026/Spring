
 Application Scope
=> Server가 시작될 때부터 종료될 때까지의 범위
   Application Scope 동안 사용할 수 있는 메모리 영역이 만들어지고
   ServletContext라는 클래스 타입의 객체로 관리됨
   
   ServletContext에 저장된 data나 객체는
   Server가 종료되기 전까지,
   웹브라우저 별로 별도의 메모리 공간을 사용하는것이 아니고
   같은 메모리 공간을 사용함
   
   
 ServletContext
   Controller에서 주입받음
=> HttpServletRequest 객체의 getServletContext()메소드를 호출해서 생성한다.


 Application Scope
=> Bean을 정의할 때 application scope로 정의하면
   Server가 시작할 대 자동으로 주입됨
   주입만 이루어지고 자동으로 application 영역에서 저장되지는 않음
   이 경우는 Server가 시작할 때 자동으로 주입되므로
   @Lazy 어노테이션을 설정하지 않아도 됨
   
   Java 설정 방식 : @ApplicationScope 으로 설정함
   Xml 설정 방식 : scope="application"으로 설정함
